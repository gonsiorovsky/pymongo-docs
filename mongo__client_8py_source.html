79  """Client for a MongoDB instance, a replica set, or a set of mongoses.
80 
81  The client object is thread-safe and has connection-pooling built in.
82  If an operation fails because of a network error,
83  :class:`~pymongo.errors.ConnectionFailure` is raised and the client
84  reconnects in the background. Application code should handle this
85  exception (recognizing that the operation failed) and then continue to
86  execute.
87 
88  The `host` parameter can be a full `mongodb URI
89  <http://dochub.mongodb.org/core/connections>`_, in addition to
90  a simple hostname. It can also be a list of hostnames or
91  URIs. Any port specified in the host string(s) will override
92  the `port` parameter. If multiple mongodb URIs containing
93  database or auth information are passed, the last database,
94  username, and password present will be used. For username and
95  passwords reserved characters like ':', '/', '+' and '@' must be
96  escaped following RFC 2396.
97 
98  :Parameters:
99  - `host` (optional): hostname or IP address of the
100  instance to connect to, or a mongodb URI, or a list of
101  hostnames / mongodb URIs. If `host` is an IPv6 literal
102  it must be enclosed in '[' and ']' characters following
103  the RFC2732 URL syntax (e.g. '[::1]' for localhost)
104  - `port` (optional): port number on which to connect
105  - `document_class` (optional): default class to use for
106  documents returned from queries on this client
107  - `tz_aware` (optional): if ``True``,
108  :class:`~datetime.datetime` instances returned as values
109  in a document by this :class:`MongoClient` will be timezone
110  aware (otherwise they will be naive)
111  - `connect` (optional): if ``True`` (the default), immediately
112  begin connecting to MongoDB in the background. Otherwise connect
113  on the first operation.
114 
115  | **Other optional parameters can be passed as keyword arguments:**
116 
117  - `maxPoolSize` (optional): The maximum number of connections
118  that the pool will open simultaneously. If this is set, operations
119  will block if there are `maxPoolSize` outstanding connections
120  from the pool. Defaults to 100.
121  - `socketTimeoutMS`: (integer or None) Controls how long (in
122  milliseconds) the driver will wait for a response after sending an
123  ordinary (non-monitoring) database operation before concluding that
124  a network error has occurred. Defaults to ``None`` (no timeout).
125  - `connectTimeoutMS`: (integer or None) Controls how long (in
126  milliseconds) the driver will wait during server monitoring when
127  connecting a new socket to a server before concluding the server
128  is unavailable. Defaults to ``20000`` (20 seconds).
129  - `serverSelectionTimeoutMS`: (integer) Controls how long (in
130  milliseconds) the driver will wait to find an available,
131  appropriate server to carry out a database operation; while it is
132  waiting, multiple server monitoring operations may be carried out,
133  each controlled by `connectTimeoutMS`. Defaults to ``30000`` (30
134  seconds).
135  - `waitQueueTimeoutMS`: (integer or None) How long (in milliseconds)
136  a thread will wait for a socket from the pool if the pool has no
137  free sockets. Defaults to ``None`` (no timeout).
138  - `waitQueueMultiple`: (integer or None) Multiplied by maxPoolSize
139  to give the number of threads allowed to wait for a socket at one
140  time. Defaults to ``None`` (no limit).
141  - `socketKeepAlive`: (boolean) Whether to send periodic keep-alive
142  packets on connected sockets. Defaults to ``False`` (do not send
143  keep-alive packets).
144 
145  | **Write Concern options:**
146  | (Only set if passed. No default values.)
147 
148  - `w`: (integer or string) If this is a replica set, write operations
149  will block until they have been replicated to the specified number
150  or tagged set of servers. `w=<int>` always includes the replica set
151  primary (e.g. w=3 means write to the primary and wait until
152  replicated to **two** secondaries). Passing w=0 **disables write
153  acknowledgement** and all other write concern options.
154  - `wtimeout`: (integer) Used in conjunction with `w`. Specify a value
155  in milliseconds to control how long to wait for write propagation
156  to complete. If replication does not complete in the given
157  timeframe, a timeout exception is raised.
158  - `j`: If ``True`` block until write operations have been committed
159  to the journal. Cannot be used in combination with `fsync`. Prior
160  to MongoDB 2.6 this option was ignored if the server was running
161  without journaling. Starting with MongoDB 2.6 write operations will
162  fail with an exception if this option is used when the server is
163  running without journaling.
164  - `fsync`: If ``True`` and the server is running without journaling,
165  blocks until the server has synced all data files to disk. If the
166  server is running with journaling, this acts the same as the `j`
167  option, blocking until write operations have been committed to the
168  journal. Cannot be used in combination with `j`.
169 
170  | **Replica set keyword arguments for connecting with a replica set
171  - either directly or via a mongos:**
172 
173  - `replicaSet`: (string or None) The name of the replica set to
174  connect to. The driver will verify that all servers it connects to
175  match this name. Implies that the hosts specified are a seed list
176  and the driver should attempt to find all members of the set.
177  Defaults to ``None``.
178  - `read_preference`: The read preference for this client. If
179  connecting directly to a secondary then a read preference mode
180  *other* than PRIMARY is required - otherwise all queries will throw
181  :class:`~pymongo.errors.AutoReconnect` "not master".
182  See :class:`~pymongo.read_preferences.ReadPreference` for all
183  available read preference options. Defaults to ``PRIMARY``.
184 
185  | **SSL configuration:**
186 
187  - `ssl`: If ``True``, create the connection to the server using SSL.
188  Defaults to ``False``.
189  - `ssl_keyfile`: The private keyfile used to identify the local
190  connection against mongod. If included with the ``certfile`` then
191  only the ``ssl_certfile`` is needed. Implies ``ssl=True``.
192  Defaults to ``None``.
193  - `ssl_certfile`: The certificate file used to identify the local
194  connection against mongod. Implies ``ssl=True``. Defaults to
195  ``None``.
196  - `ssl_cert_reqs`: Specifies whether a certificate is required from
197  the other side of the connection, and whether it will be validated
198  if provided. It must be one of the three values ``ssl.CERT_NONE``
199  (certificates ignored), ``ssl.CERT_OPTIONAL``
200  (not required, but validated if provided), or ``ssl.CERT_REQUIRED``
201  (required and validated). If the value of this parameter is not
202  ``ssl.CERT_NONE`` and a value is not provided for ``ssl_ca_certs``
203  PyMongo will attempt to load system provided CA certificates.
204  If the python version in use does not support loading system CA
205  certificates then the ``ssl_ca_certs`` parameter must point
206  to a file of CA certificates. Implies ``ssl=True``. Defaults to
207  ``ssl.CERT_REQUIRED`` if not provided and ``ssl=True``.
208  - `ssl_ca_certs`: The ca_certs file contains a set of concatenated
209  "certification authority" certificates, which are used to validate
210  certificates passed from the other end of the connection.
211  Implies ``ssl=True``. Defaults to ``None``.
212  - `ssl_match_hostname`: If ``True`` (the default), and
213  `ssl_cert_reqs` is not ``ssl.CERT_NONE``, enables hostname
214  verification using the :func:`~ssl.match_hostname` function from
215  python's :mod:`~ssl` module. Think very carefully before setting
216  this to ``False`` as that could make your application vulnerable to
217  man-in-the-middle attacks.

219  .. mongodoc:: connections
220 
221  .. versionchanged:: 3.0
222  :class:`~pymongo.mongo_client.MongoClient` is now the one and only
223  client class for a standalone server, mongos, or replica set.
224  It includes the functionality that had been split into
225  :class:`~pymongo.mongo_client.MongoReplicaSetClient`: it can connect
226  to a replica set, discover all its members, and monitor the set for
227  stepdowns, elections, and reconfigs.
228 
229  The :class:`~pymongo.mongo_client.MongoClient` constructor no
230  longer blocks while connecting to the server or servers, and it no
231  longer raises :class:`~pymongo.errors.ConnectionFailure` if they
232  are unavailable, nor :class:`~pymongo.errors.ConfigurationError`
233  if the user's credentials are wrong. Instead, the constructor
234  returns immediately and launches the connection process on
235  background threads.
236 
237  Therefore the ``alive`` method is removed since it no longer
238  provides meaningful information; even if the client is disconnected,
239  it may discover a server in time to fulfill the next operation.
240 
241  In PyMongo 2.x, :class:`~pymongo.MongoClient` accepted a list of
242  standalone MongoDB servers and used the first it could connect to::
243 
244  MongoClient(['host1.com:27017', 'host2.com:27017'])
245 
246  A list of multiple standalones is no longer supported; if multiple
247  servers are listed they must be members of the same replica set, or
248  mongoses in the same sharded cluster.
249 
250  The behavior for a list of mongoses is changed from "high
251  availability" to "load balancing". Before, the client connected to
252  the lowest-latency mongos in the list, and used it until a network
253  error prompted it to re-evaluate all mongoses' latencies and
254  reconnect to one of them. In PyMongo 3, the client monitors its
255  network latency to all the mongoses continuously, and distributes
256  operations evenly among those with the lowest latency. See
257  :ref:`mongos-load-balancing` for more information.
258 
259  The ``connect`` option is added.
260 
261  The ``start_request``, ``in_request``, and ``end_request`` methods
262  are removed, as well as the ``auto_start_request`` option.
263 
264  The ``copy_database`` method is removed, see the
265  :doc:`copy_database examples </examples/copydb>` for alternatives.
266 
267  The :meth:`MongoClient.disconnect` method is removed; it was a
268  synonym for :meth:`~pymongo.MongoClient.close`.
269 
270  :class:`~pymongo.mongo_client.MongoClient` no longer returns an
271  instance of :class:`~pymongo.database.Database` for attribute names
272  with leading underscores. You must use dict-style lookups instead::
273 
274  client['__my_database__']
275 
276  Not::
277 
278  client.__my_database__

374  def _cache_credentials(self, source, credentials, connect=False):
375  """Save a set of authentication credentials.
376 
377  The credentials are used to login a socket whenever one is created.
378  If `connect` is True, verify the credentials on the server first.
379  """
380  # Don't let other threads affect this call's data.
381  all_credentials = self.__all_credentials.copy()
382 
383  if source in all_credentials:
384  # Nothing to do if we already have these credentials.
385  if credentials == all_credentials[source]:
386  return
387  raise OperationFailure('Another user is already authenticated '
388  'to this database. You must logout first.')
389 
390  if connect:
391  server = self._get_topology().select_server(
392  writable_preferred_server_selector)
393 
394  # get_socket() logs out of the database if logged in with old
395  # credentials, and logs in with new ones.
396  with server.get_socket(all_credentials) as sock_info:
397  sock_info.authenticate(credentials)
398 
399  # If several threads run _cache_credentials at once, last one wins.
400  self.__all_credentials[source] = credentials
401 
402  def _purge_credentials(self, source):
403  """Purge credentials from the authentication cache."""
404  self.__all_credentials.pop(source, None)

434  """Purge an index from the index cache.
435 
436  If `index_name` is None purge an entire collection.

831  def kill_cursors(self, cursor_ids, address=None):
832  """Send a kill cursors message soon with the given ids.
833 
834  Raises :class:`TypeError` if `cursor_ids` is not an instance of
835  ``list``.
836 
837  This method may be called from a :class:`~pymongo.cursor.Cursor`
838  destructor during garbage collection, so it isn't safe to take a
839  lock or do network I/O. Instead, we schedule the cursor to be closed
840  soon on a background thread.
841 
842  :Parameters:
843  - `cursor_ids`: list of cursor ids to kill
844  - `address` (optional): (host, port) pair of the cursor's server.
845  If it is not provided, the client attempts to close the cursor on
846  the primary or standalone, or a mongos server.
847 
848  .. versionchanged:: 3.0
849  Now accepts an `address` argument. Schedules the cursors to be
850  closed on a background thread instead of sending the message
851  immediately.
852  """
853  if not isinstance(cursor_ids, list):
854  raise TypeError("cursor_ids must be a list")
855 
856  # "Atomic", needs no lock.
857  self.__kill_cursors_queue.append((address, cursor_ids))
858 
859  # This method is run periodically by a background thread.
860  def _process_kill_cursors_queue(self):
861  """Process any pending kill cursors requests."""
862  address_to_cursor_ids = defaultdict(list)
863 
864  # Other threads or the GC may append to the queue concurrently.
865  while True:
866  try:
867  address, cursor_ids = self.__kill_cursors_queue.pop()
868  except IndexError:
869  break
870 
871  address_to_cursor_ids[address].extend(cursor_ids)
872 
873  # Don't re-open topology if it's closed and there's no pending cursors.
874  if address_to_cursor_ids:
875  topology = self._get_topology()
876  for address, cursor_ids in address_to_cursor_ids.items():
877  try:
878  if address:
879  server = topology.select_server_by_address(address)
880  else:
881  # Application called close_cursor() with no address.
882  server = topology.select_server(
883  writable_server_selector)
884 
885  server.send_message(message.kill_cursors(cursor_ids),
886  self.__all_credentials)
887  except ConnectionFailure as exc:
888  warnings.warn("couldn't close cursor on %s: %s"
889  % (address, exc))
